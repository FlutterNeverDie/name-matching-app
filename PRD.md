# 📑 [이름으로 보는 궁합] 제품 요구 사양서 (PRD)

## 1. 프로젝트 개요

- **서비스 명**: 이름으로 보는 궁합
- **목적**: 두 사람의 이름을 입력받아 연인, 친구, 사업 카테고리별 운명 점수(0~10)를 산출하는 결정론적 웹앱 도구.
- **핵심 가치**: "심플한 입력, 확실한 결과, 연속적인 재미."

---

## 2. UI/UX 원칙

- **Single Screen Web App**: 화면 전환 시 스크롤 없이 고정된 레이아웃을 유지한다.
- **Minimalist Aesthetic**: 비비드 퍼플(#8E2DE2)과 핫 핑크(#FF416C) 그라데이션 배경에 화이트 요소를 배치한 신비로운 무드.
- **Sticky Input**: '다른 이름이랑 하기' 기능을 통해 사용자 경험의 단절을 최소화한다.

---

## 3. 상세 기능 요구사항

### A. 메인 화면 (Input Stage)

- **내 이름 입력란**: 최대 10자 제한.
- **상대 이름 입력란**: 최대 10자 제한.
- **카테고리 선택**: [연인 / 친구 / 사업] 3개 탭 구성 (기본값: 연인).
- **결과 보기 버튼**: 입력값이 모두 존재할 때 활성화.

### B. 운명 지수 산출 로직 (Deterministic Logic)

- **입력값**: nameA, nameB, category
- **로직 단계**:
  1. nameA와 nameB를 합친 문자열의 모든 유니코드 값을 합산한다.
  2. category별 고정 가중치(Salt)를 더한다. (연인: +7, 친구: +3, 사업: +1)
  3. 최종 합산 값을 11로 나눈 나머지(% 11)를 구한다.
- **결과값**: 무조건 0에서 10 사이의 정수가 도출되어야 하며, 같은 입력에는 항상 같은 값이 나와야 한다.

### C. 결과 화면 (Result Stage)

- **점수 표시**: 중앙에 0~10 사이의 점수를 크게 노출.
- **궁합 텍스트**: 점수 범위에 따른 결과 노출.
  - 0~3 (나쁨): 차분한 그레이톤 문구 ("조금 더 노력이 필요해요")
  - 4~6 (보통): 화이트톤 문구 ("무난하고 평범한 사이")
  - 7~10 (최고): 골드톤 및 화려한 효과 ("하늘이 내린 찰떡궁합")
- **중앙 컴포넌트**: 내 이름 [아이콘] 상대 이름 표시.
  - 아이콘은 카테고리에 따라 변경 (연인: ❤️, 친구: 🤝, 사업: 💼).

### D. 네비게이션 및 액션 (Navigation & Action)

- **다른 이름이랑 하기 (Contextual Button)**:
  - 위치: 결과 컴포넌트 바로 하단.
  - 동작: 클릭 시 메인으로 이동하되 '내 이름'은 유지, '상대 이름'만 초기화 후 포커스(Focus) 처리.
- **바텀 네비게이션 (Bottom Nav)**:
  - **다시 만들기**: 클릭 시 모든 입력값(내 이름, 상대 이름)을 초기화하고 메인 화면으로 이동.
  - **공유하기**: Web Share API를 호출하여 서비스 URL 공유 (단순 앱 홍보).

---

## 4. 기술적 제어 규칙

- **Zero Server-side**: 모든 계산은 클라이언트 사이드(Browser)에서 즉시 처리하며 데이터를 저장하지 않는다.
- **Validation**: 이름 미입력 시 버튼 비활성화 또는 경고 메시지 노출.
- **Responsive**: 모바일 브라우저 뷰포트(100vh)에 최적화하여 요소가 화면 밖으로 나가지 않게 제어한다.
